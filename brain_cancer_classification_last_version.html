7. PCA — Dimensionality Reduction¶

54,000 genes is far too many for a classifier. PCA compresses them into 50 principal components that capture the most variation.

Think of it as finding the 50 most informative "directions" in the data.

In [25]:
pca = PCA(n_components=50, random_state=42)
X_train_pca = pca.fit_transform(X_train_scaled)
X_test_pca  = pca.transform(X_test_scaled)

explained = pca.explained_variance_ratio_.cumsum()[-1] * 100
print(f"50 components explain {explained:.1f}% of the original variance")
print(f"Shape before PCA: {X_train_scaled.shape}")
print(f"Shape after PCA:  {X_train_pca.shape}")

50 components explain 87.1% of the original variance
Shape before PCA: (104, 54675)
Shape after PCA:  (104, 50)

Why Try Multiple Models?¶

There is no single 'best' model for every dataset. Different algorithms make different assumptions about the data, so the only way to know which one works best for your problem is to run them all and compare.

We will now test two more classifiers on the same PCA-reduced data:

  • Decision Tree — easy to interpret, makes yes/no splits on features
  • K-Nearest Neighbors (KNN) — classifies each patient based on its closest neighbors

All three models use the exact same X_train_pca / X_test_pca so the comparison is fair.

12. K-Nearest Neighbors (KNN) Classifier¶

  • n_neighbors=5 → look at the 5 closest training samples
  • metric='euclidean' → measure distance in PCA space

KNN is very sensitive to scale, which is why scaling + PCA first was important.
